Git使用教程

发表于2020-03-02|更新于2024-02-15|git
|阅读量:

Git是目前最为流行的分布式版本控制系统，其运行原理图如下：

  • Remote: 远程仓库
  • Respository: 本地仓库
  • Index / Stage：暂存区
  • workspace：工作区
    流程： 在workspace中做编程开发，然后add需要提交的文件到Index中，确认无误后commit到Respository,再push同步到Remote。

创建 & 拉取仓库

Git的配置

Git的设置文件为.gitconfig，它可以在用户主目录下（全局配置），也可以在项目目录下（项目配置）。

此外还可以配置.gitignore文件，一般在项目目录下面（或者子目录），主要是用来忽略对项目当中某些文件的跟踪。 在.gitignore中直接输入对应的文件名或目录名称即可。注意，若该文件已经在远程仓库中，建议先git rm 来删除掉 远程仓库的文件，然后再更新.gitignore文件！

分支命名

如果是需求分支，规范为：feature/分支名，

例如 feature/WeCar_20180806_v2.9.0_new_ui

如果是修复bug，建议也是拉分支，规范为： bugfix/分支名/bug名称，

例如 bugfix/WeCar_20180806_v2.9.0_new_ui/adapter
